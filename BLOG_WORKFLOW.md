# 📝 네이버 블로그 패션 정보 포스팅 자동화 워크플로우

이 문서는 AI가 동일한 양식과 퀄리티로 연예인 패션 정보 블로그 포스팅을 자동 작성하기 위해 지켜야 할 워크플로우를 정리한 가이드입니다. 
다른 컴퓨터나 새로운 세션에서 작업을 시작하더라도 이 문서를 읽으면 전체 진행 과정을 명확하게 파악할 수 있습니다.

## 전체 진행 과정 (Workflow)

### 1. 준비물 전달 및 명령 (사용자 -> AI)
사용자는 다음 정보를 AI에게 전달하며 포스팅 작성을 지시합니다.
*   **연예인 착장 SNS 링크** (예: 인스타그램 게시물 링크)
*   **착장 제품 구매 링크** (예: 무신사, 공홈, 지그재그 링크 등)
*   (선택) 특별히 강조하고 싶은 코멘트나 해시태그
> **💡 프롬프트 예시:** *"BLOG_WORKFLOW.md 문서의 워크플로우대로 블로그 글 작성해줘. 연예인 사진 링크는 [이거]고, 제품 링크는 [이거랑 이거]야."*

### 2. 스타일 데이터베이스 열람 (AI 자체 수행)
*   AI는 글을 작성하기 전, 반드시 같은 폴더에 있는 **`BLOG_STYLE_DATABASE.md`** 파일을 먼저 읽고 숙지합니다.
*   해당 파일에는 블로그 특유의 친근한 말투(`ㅎㅎ`, `:)`, `..!` 등), 금지된 이모티콘(기계적인 이모지), 규격화된 포맷 가이드라인, 그리고 이전에 작성했던 우수 포스팅 샘플들이 누적되어 있습니다.

### 3. 제품 정보 자동 추출 (AI 자체 수행)
*   AI는 `app.py` (Universal Grabber) 서버를 백그라운드에서 실행합니다.
*   전달받은 제품 링크들을 자체 API(`/api/fetch_product`)로 전송하여 **브랜드명, 제품명, 가격** 등의 데이터를 스크래핑합니다.
*   오류가 발생하거나 인코딩 문제가 생기면 AI가 직접 스크립트를 작성하여 정확한 한국어 정보를 확보합니다.

### 4. 감성 텍스트 작성 및 HTML 생성 (AI -> 사용자)
*   `BLOG_STYLE_DATABASE.md`의 스타일 규칙에 맞춰 추출된 정보를 바탕으로 네이버 블로그 감성의 텍스트를 작성합니다.
*   단순 텍스트 복사 시 발생하는 폰트 깨짐 현상을 방지하기 위해, 복사&붙여넣기만으로 스마트에디터 서식이 100% 유지되는 **HTML 파일(`blog_post.html`)**을 생성해 작업 폴더에 저장합니다. (폰트 크기 15px, 줄간격 1.8, 좌측 정렬 등)

### 5. 블로그 발행 (사용자)
*   사용자는 생성된 `blog_post.html` 파일을 브라우저로 열고 `Ctrl + A` -> `Ctrl + C` 로 복사합니다.
*   네이버 블로그 에디터에 `Ctrl + V` 로 붙여넣고, `[ 사진 삽입 위치 ]` 텍스트 자리에 실제 사진만 교체하여 넣은 후 발행합니다.

### 6. 완성본 누적 대기 및 업데이트 (사용자 -> AI)
*   글 작성을 마치고 HTML 파일 생성이 완료되더라도, 곧바로 데이터베이스에 추가하지 않고 대기합니다.
*   사용자가 최종 결과물을 확인하고 **"데이터베이스에 넣어줘"**라고 명시적으로 요청할 경우에만, 방금 작성한 블로그 글의 최종 완성 텍스트를 **`BLOG_STYLE_DATABASE.md` 하단의 '누적 작성글 데이터베이스' 영역에 추가**합니다.
*   수정 요청이 있을 경우, 수정이 완료된 최종본을 기준으로 승인 시점에 누적시킵니다.
